

# 현재 프로젝트 상태 확인 결과

현재 프로젝트는 **학원/교육 업종 웹 제작 시스템**입니다. 부동산/공인중개 시스템이 아닙니다.

확인된 근거:
- `brief-schema.ts`: `EducationSubtype` (보습, 입시, 영어/어학 등), `SiteType` (상담전환형, 과정탐색형, 성과신뢰형 등)
- `brief-engine.ts`: 강사진, 커리큘럼, 수강 후기, 성과/합격 사례 기반 로직
- `seo-config.ts`: "학원/교육" 제목/설명 전체
- `navigation.ts`: 교육 업종 가이드 메뉴 구조
- `package.json`: `name: "edu-guide-system"`

**부동산/공인중개 보강 프롬프트를 이 프로젝트에 적용하려면**, 스키마(`ClientBriefData`), 엔진(`brief-engine.ts`), SEO 설정, 네비게이션, 모든 가이드 페이지를 부동산 업종으로 전면 교체해야 합니다. 이는 기존 교육 시스템을 파괴합니다.

---

## 선택이 필요합니다

어떻게 진행할지 결정해 주세요:

1. **이 프로젝트를 부동산/공인중개 시스템으로 전환** — 교육 관련 코드를 모두 부동산용으로 교체
2. **교육 시스템 유지 + 부동산은 별도 프로젝트** — 현재 프로젝트는 교육 마감에 집중
3. **멀티 업종 시스템으로 확장** — 업종 선택 UI를 추가하고 교육/부동산 데이터를 모두 포함

결정 후 바로 구현 계획을 제시하겠습니다.
